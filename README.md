# AIRSOFT FPS — BB弾道リアルシミュレーション搭載FPS

three.js製のエアソフト(電動ガン)FPSゲーム。BB弾の弾道は
[adspawn/bb-ballistics](https://github.com/adspawn/bb-ballistics) の物理モデルを
3Dに拡張して移植しています。

## 起動方法

ESモジュール使用のためローカルサーバーが必須です(初回のみ依存パッケージの
インストールが必要)。

```
npm install
npm start
```

→ ブラウザで http://localhost:8377 を開く
(three.js・PeerJS は `libs/` に同梱、オフラインで動作。
`server.js` は静的ファイル配信のみを行う軽量サーバーで、
`npx http-server` 等の単純な静的サーバーでも代用できます。
オンラインPVPの通信自体はサーバーを介さないP2P(WebRTC)のため、
ゲームを公開する場合も静的ホスティングだけで足ります)

## コード構成

`index.html` はHTML/CSS/DOM構造のみ(スタイルは `styles.css` に分離)。
ゲームロジックは `src/` 以下のESモジュールに機能ごとに分割されています。

| ファイル | 役割 |
|---|---|
| `src/state.js` | three.jsシーン初期化・共有state(`S`設定, `player`, `pvp`, `RT`実行時状態など) |
| `src/physics.js` | BB弾道の物理モデル(抗力・マグヌス・2Dシミュレーション) |
| `src/bb.js` | BB弾プール・弾道更新・跳弾・当たり判定(命中処理はコールバック登録方式) |
| `src/sound.js` / `src/effects.js` | WebAudio効果音 / パーティクル・画面メッセージ |
| `src/targets.js` | 射撃練習場のターゲット・タイムアタック |
| `src/gun.js` | 銃モデル(GLTF)・アイアンサイト校正・サイト調整デバッグモード |
| `src/player.js` | プレイヤー移動・射撃・キーボード/マウス入力配線 |
| `src/bots.js` | 対戦(NPC)のAI・バリケード配置・被弾死亡演出(PVPと共有)。移動は[Yuka](https://mugen87.github.io/yuka/)の操舵行動(到着+障害物回避+NPC間分離、`libs/yuka.module.js`に同梱)、状態遷移・射撃は自前FSM |
| `src/mapEditor.js` | マップ作成モード |
| `src/p2p.js` | P2P通信層([PeerJS](https://peerjs.com/)/WebRTCの接続管理・メッセージ送受信のみ。ゲームロジックは持たない) |
| `src/pvp.js` | オンラインPVP(P2Pロビー・ホスト権威の部屋/勝敗判定・NPC・チーム制) |
| `src/menu.js` | メインメニュー設定・モード切替(`applyMode`) |
| `src/main.js` | 全モジュールの読込み・配線・メインループ起動 |

モジュール間の共有可変状態は基本的に `state.js` のオブジェクト(`S`, `player`, `pvp`,
`RT` など)を各モジュールが直接書き換える形で連携しています(ESモジュールの
ライブバインディングにより、プロパティ変更は自動的に他モジュールへ反映されます)。

## 銃の種類

メニューの「銃の種類」から3種類を選べます(装備の弾道設定は全銃共通)。

| 銃 | 射撃モード | 特徴 |
|---|---|---|
| アサルトライフル | セミ / フルオート | 標準。アイアンサイトでADS |
| ショットガン | セミオートのみ | **1トリガーで3発同時発射**。散りが大きく近距離向き |
| スナイパーライフル | セミオートのみ | ADSで**スコープ表示**(高倍率・円形ケラレ+レティクル)。散りが最小 |

サイト調整(ピッチ/ヨー/マズル位置/保持位置/散り具合など)は**銃ごとに独立して保存**されます。

## ビューモデル

一人称視点の銃は `assets/` のGLBモデル(AR-15 / ショットガン / ハンティングライフル)を
GLTFLoaderで読み込んで表示。読込み完了までは簡易プロシージャル銃を
フォールバック表示し、成功次第切り替わります。実頂点位置を検証したうえで
実銃相当の全長へ自動スケーリング・原点合わせを行い、モデルごとに向きが異なるため
**最長軸をZ(前後)へ揃え、銃口が前方(-Z)を向くよう自動で回転補正**します。

## 物理モデル

```
並進:  m r'' = -mg e_z - ½Cd ρπR² |v|v + Cl(4/3)πR³·2ρ (ω × v)
回転:  I ω'  = -½Cf ρR²(8πR²/3)·√(v² + (cRω)²)·ω
```

- **重力 + 空気抵抗 + マグヌス揚力**(ホップアップのバックスピンによる浮力)を数値積分
- **風**: 空気力(抗力・マグヌス力)は対地速度ではなく**対気速度(弾速 − 風速)**で評価するため、
  横風ではBBが風下へ流され、追い風/向かい風では飛距離が伸縮します
  (0.20g・初速92m/s で横風10m/s の場合、約44m先で7m以上流されます)
- 抗力係数 Cd: **Loth (2021)** モデル(Cd≈0.45)
- 回転は空気摩擦(Cf = 1.328/√Re)で徐々に減衰 → 遠距離で弾道が落ちる
- 6mm BB弾 (d=5.95mm)、空気密度 1.205 kg/m³、Cl=0.12
- ゲーム内は dt=1ms の固定サブステップ積分(参照実装 dt=0.1ms との着弾距離誤差 <0.1% を検証済み)

## ホップアップ調整

- **ホイール / [ ]** でリアルタイムに回転数(rps)を調整
- **O キー**で「適正ホップ」(射線からの浮き上がり5mm以内で最大飛距離、二分探索で算出)に自動調整
- HUDにゼロイン距離・最大飛距離をリアルタイム表示
- ホップを掛けすぎると実銃同様に鬼ホップ(浮き上がり)します

## サイト調整モード（デバッグ）

通常プレイの邪魔にならないよう**メインメニューには表示されません**。
`http://localhost:8377/?sightcal` のように**URLに `?sightcal` を付けてアクセス**すると
直接このモードで起動し、AR-15モデルのアイアンサイト・BB弾の発射方向・マズル位置を
実射しながら追い込めます。何も調整しない状態では、実測済みの固定キャリブレーション値
(ピッチ-0.7°/ヨー-0.6°/ロール0°、ゼロイン(30,0)px、倍率25°、マズル(-0.002,-0.007,0.357)m、
保持位置(0.235,-0.225,-0.35)m)がゲーム全体の既定値として使われます。

- **ピッチ/ヨー/ロール**: フロントサイトを画面中心に固定したまま、リアサイト側を
  ピボット回転で動かして視覚的に照準を合わせる
- **照準方向のズレ(ゼロイン調整)**: ADS時に実際にBB弾が飛ぶ方向そのもの
  (画面中心=通常の照準、ずらすとサイト越しの照準点と着弾方向にズレを持たせられる)。
  腰だめ撃ち(非ADS)には影響しません — 腰だめ時はカメラ中心からレイキャストして
  最初に当たる点(障害物/地面/的/NPC)を着弾点とし、マズルからその点へ向けて発射する
  **動的ゼロイン**方式で、マズルの左右オフセットによる視差を補正してクロスヘアどおりに着弾します
- **マズル(発射始点)のXYZ**: オレンジ球で可視化、右ドラッグで自由視点に回り込んで確認可能
- **クロスヘアのサイズ**: 十字の長さ・丸の半径を調整(丸は既定0=非表示)
- **銃の保持位置(腰だめ描写)のXYZ**: 一人称視点での銃の表示位置
- **BB弾の散り具合**: 発射方向の円錐拡散(半頂角・度)を腰だめ/ADSそれぞれで調整。
  実際の拡散は移動中1.8倍・しゃがみ0.7倍で変化します
- パネル上部の**銃の種類**チップで調整対象の銃を切り替えられ、**値は銃ごとに別々に保存**されます
  (「リセット」はその銃の既定値に戻ります)
- **🚶 WASD移動で確認**: サイト調整パネルを開いたままWASDで歩き回り、右ドラッグで視点回転・
  左クリックで試射しながら、腰だめ描写の銃を見て保持位置を追い込める切替モード
  (ポインタロックを使わないため、移動中もスライダーをそのまま操作できる)
- 各項目はスライダーと数値入力の両方で調整でき、**JSONで書き出し/読み込み**が可能
  (「確定・保存」でブラウザに永続保存、JSONファイルは別環境への持ち出し用)。
  保存した値は射撃練習・対戦・PVPを含む全モードに反映されます

## モード

- **射撃練習**: スチールプレート・マンターゲット(頭2倍)・空き缶(吹き飛ぶ)・
  ムーバーを撃つレンジ。Tキーで60秒タイムアタック。
  - **着弾表示**: 撃つたびにHUDへ**着弾距離・着弾時のエネルギー(J)・残速**を表示します
    (的への命中だけでなく、地面や障害物への着弾・失速も対象)。
    初速時のエネルギー(メニュー表示)と比べることで、距離によるエネルギー減衰が確認できます。
  - **風**: 風速(0〜15 m/s)と風向(方位)を設定できます。風向は風が吹いて**いく**方位で、
    射座は北(-Z)向きのため 0°=追い風 / 180°=向かい風 / 90°=右へ流す横風 になります。
  - **吹き流し**: 射場に2本設置してあり(手前左・奥右)、風向と風速を視覚的に確認できます。
    実物と同じく**風下へなびき、風が弱いほど垂れ下がり**(12m/sで水平)、
    風速に応じて揺らぎます。
- **対戦(NPC)**: ドラム缶・ベニア板(0.9×1.8m縦型)・木箱が配置されたフィールドで
  NPCと戦うオフライン対戦。**ルールとUIはオンラインPVPと統一**されており、
  下記3種類から選べます(**被弾は即死(ゲームオーバー)**、撃破されたNPCは復活しません)。
  - **バトルロワイアル**: チーム無し＝全員が敵(NPC同士も撃ち合う)。
    敵NPCを全滅させれば勝利。NPCの数・強さは単一設定。
  - **殲滅戦**: プレイヤーは🔴赤チーム。敵の🔵青NPCを全滅させれば勝利。
  - **フラッグ戦**: 🔴赤/🔵青の2チーム制。敵陣(青)の旗に到達すれば勝利、
    自陣(赤)の旗を青NPCに奪われると敗北。味方の赤NPCが敵陣の旗を取っても勝利です。
  - 殲滅戦・フラッグ戦では**NPCの数と強さを🔴赤(味方)/🔵青(敵)ごとに個別設定**でき、
    味方への誤射は無効(プレイヤー・NPCとも)。NPCはチームカラーで表示されます。
  - 強さは よわい/ふつう/つよい の3段階(命中精度・反応速度・連射・リード射撃が変化)。
    フラッグ戦では強いNPCほど貪欲に敵陣フラッグを狙いに来ます(直行確率 25%/45%/70%)。
  - NPCの移動はYukaの操舵行動ベースで、バリケードを滑らかに回避し、NPC同士も重なりません。
  - バリケード配置は「ランダム」(毎試合42個自動生成)と「カスタムマップ」
    (マップ作成モードで保存したもの)を選択可能。
- **マップ作成モード**: 対戦フィールドのバリケードを自分で設置。
  1/2/3キーでベニア板/ドラム缶/木箱を選択、**ホイール(またはR)で45°刻みに回転**、
  左クリックで設置、右クリックで削除(緑=設置可・スナップ中/水色=設置可・フリー配置/赤=設置不可)。
  **クリエイティブ飛行**(Space上昇/Shift下降・重力なし)で自由に動き回れます。
  **Fキーでオブジェクトスナップの ON/OFF を切り替え**
  (積み重ね中心/隣接/0.25mグリッド/縁吸着)。OFFならレイの着弾点にそのまま自由配置できます。
  **E/Iキー**でJSONファイルへの書き出し/読み込みも可能(localStorageへの自動保存とは別に、
  ファイルとして保存・共有できます)。対戦モードで「カスタムマップ」を選ぶと使用されます
  (積んだ高さも衝突判定に反映)。
- **オンラインPVP**: [PeerJS](https://peerjs.com/)(WebRTC)によるサーバーレスP2P対人戦。
  シグナリングのみPeerJSの無料クラウドサーバーを使い、部屋作成後の通信(位置・射撃・
  勝敗判定など)はブラウザ同士が直接WebRTCで行うため、**対戦中は中継サーバーを一切
  経由しません**(通信費0円)。
  - **接続方式(URL共有型)**: 「部屋を作る」を押すと `room-xxxxxxx` 形式の短いIDを
    PeerJSのシグナリングサーバーから取得し、`https://.../?room=room-xxxxxxx` の
    招待URLが表示されます(「コピー」ボタンでクリップボードへ)。これを友達に送ると、
    URLを開いた瞬間にページがURLから部屋IDを自動で読み取り、PeerJSで直接接続
    (WebRTC)を試みます — 部屋一覧からの選択は無く、URL(またはロビー画面の
    「コードで参加」欄への手入力)で参加します。
  - **スター型トポロジー**: ゲストはホストとだけ接続し、ゲスト間で共有すべき情報
    (他プレイヤーの位置・発射など)はホストが中継します。ロビー(部屋管理)・
    被弾判定・キル集計・リスポーン指示・勝敗判定はすべてホストのブラウザが
    権威的に処理します(旧サーバー実装をブラウザ内に移植したもの)。
  - 全員準備完了(ホスト自身は不要)でホストが試合開始 — **1人だけでもNPC相手に開始可能**。
    被弾すると(対戦(NPC)と同じ)三人称カメラで倒れる演出。
    弾道シミュレーションは各クライアントで実行します。
  - **試合中にMキーを押すと**ポインタロックだけ解除してロビー画面に戻ります(試合は継続)。
    「🔓 試合に戻る」でポインタロックを取り直して同じ試合へシームレスに復帰できます。
  - **ゲーム種類**(部屋作成時に選択):
    - **バトルロワイアル**: チーム無し、全員が敵。リスポーン無し、生存する人間が
      1人になり、かつ**NPCも全滅した時点**でその人が勝利(全員が敵NPCのみの
      1人プレイでも、NPCを全滅させれば勝利フラグが立ちます)。
      NPCの数(0〜8体)・強さは単一設定。
    - **殲滅戦**: 🔴赤/🔵青の2チーム制(参加順で自動的に交互に振り分け)。リスポーン無し、
      片方のチームの人間プレイヤー・**NPCが両方とも全滅**した時点で相手チームの勝利。
    - **フラッグ戦**: 🔴赤/🔵青の2チーム制。敵陣の旗に到達すると即座にそのチームの勝利。
      決着がつくまで被弾してもリスポーンして戦い続けられます。
      **NPCも一定確率で敵陣の旗を狙いに行き、到達すればそのチームの勝利になります**。
    - 殲滅戦・フラッグ戦は**NPCの数と強さを🔴赤/🔵青チームごとに個別設定**可能
      (バトルロワイアルはNPCも含めて全員が敵のため単一設定のまま)。
  - **リスポーン地点**: チーム戦(殲滅戦/フラッグ戦)は自陣フラッグを中心にした同心円リング上
    (内側=プレイヤー用/外側=NPC用で分離)から選ばれるため、プレイヤーとNPCのリスポーンが
    重なりません。バトルロワイアルは従来通りフィールド内の共通スポーン地点を使用します。
  - **NPC**: ホストのクライアントのみがAI・移動・射撃を計算する「ホスト権威」方式。
    チーム戦のNPC・他プレイヤーのアバターは🔴赤/🔵青のチームカラーで表示されます。
    NPCは敵のプレイヤー/NPCのうち最も近いものを狙うため、**NPC同士も撃ち合います**
    (バトルロワイアルでは全NPCが敵同士。NPC同士の撃破はスコアに加算されません)。
    チーム戦では味方への誤射はしません(人間プレイヤー側もチーム戦では味方弾が無効化されます)。
    位置と射撃はルーム内へ中継され、他のクライアントはその情報を元にNPCを描画・
    被弾判定するのみです(ホストが退出するとNPCも消えます)。
  - **バリケード配置の同期**: ランダムでもカスタムマップでも、ホストが試合開始時に
    一度だけ配置データを確定してP2P経由で全クライアントへ配布するため、
    プレイヤーごとにバリケード位置がズレることはありません。

## 跳弾（リコシェ）

BB弾は障害物に一定以上の速度(25m/s超)で命中すると、面の法線方向に反射して
飛び続けます(最大2回・反射のたびに速度55%に減衰・回転も半減)。
メニューの**「跳弾ヒット」ON/OFF**で、跳ねた後のBB弾が命中判定を持つかどうかを
切り替えられます(OFFでも跳弾自体は起こりますが、命中はしなくなります。
直撃は常に判定されます)。

対戦(NPC)・マップ作成・オンラインPVPは、射撃練習場とは別の**専用50m×50m
フィールド**(射撃練習場から離れた場所)で行われます。

## 装備設定(メニュー画面)

メニューは選択中のモードに関係する設定だけを表示します
(射撃練習=装備設定 / 対戦(NPC)=対戦ルール・NPC設定 / オンラインPVP=ロビーで設定)。
装備の値自体は全モード共通で使われます。

- BB弾重量: 0.12g〜0.43g — 重い弾ほど直進性が高く飛距離が伸びる
- 初速: 60〜150 m/s — エネルギー表示付き(日本の法定 0.98J 超過で警告)
- サイクル: 5〜30発/秒(フルオート連射速度)
- 抗力モデルは Loth (2021, Cd≈0.45) 固定、トレーサーは常時ON
- 弾切れ時は発射ボタンでオートリロード(1.6秒)

## ゲーム内容

10m〜60mに配置されたスチールプレート+ムーバー(移動ターゲット、得点2倍)を撃つ
シューティングレンジ。得点は命中距離に応じて加算。ヒット音は距離に応じた
伝播遅延付き(50m先の的は「カン!」が約0.15秒遅れて聞こえます)。
